## 本次代码优化

### 1. 保守候选门控

- 单独出现“如果”“哪些情况”等条件词不再触发；
- Query Analysis 要求澄清时跳过；
- 新旧版本比较和明确二选一问题不执行并行分解；
- 保留多问句和强组合信号进入模型确认。

### 2. 分解收益校验

分解模型新增：

- `decomposition_type`
- `benefit_score`

只有高于配置阈值的 `PARALLEL` 分解才执行。`DEPENDENT` 查询暂时回退 M9
链路，避免把真正的顺序多跳错误地当成并行查询。

### 3. 动态 Rerank TopN

复杂查询的调用级 TopN：

```text
基础 rerank_top_n + min(子问题数量, 最大额外数量)
```

该值被真正透传到 `RerankService`、`Qwen3Reranker` 和百炼接口。此前 Graph
虽然计划保留更多候选，但底层接口仍只返回 Top5，导致第 6、7 位必要事实无法
恢复。本次修复后，同一次 Rerank 调用即可拿到扩展候选，不增加第二次调用。

### 4. 组合证据可回答性

动作模型现在允许：

- 多个 Context 证据块联合覆盖复杂问题；
- 明确特殊对象例外优先于一般规则；
- 不要求单个文档逐字描述完整组合场景。

同时增加边界：

- 不能拿相邻业务 SOP 代替用户明确询问的流程；
- 相关概述不能代替具体赔付标准、完整步骤和明确条件。
